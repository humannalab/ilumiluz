06 — Movimento

Movimento
& Interação

Movimento na Ilumiluz é contenção expressiva. Cada transição tem propósito — nada é decorativo, nada é brusco.

Filosofia de movimento

Suave

Nenhuma transição deve chamar atenção para si mesma. O movimento serve ao conteúdo — nunca o interrompe.

Intencional

Cada animação comunica algo: confirmação de interação, entrada de conteúdo, estado de carregamento. Nunca por estética pura.

Lento o suficiente

O ritmo da Ilumiluz é pausado. Transições muito rápidas (< 150ms) criam frieza. Prefira 250-400ms para gestos principais.

Curvas de easing

3 curvas essenciais

Passe o mouse sobre cada card para ver a curva em ação.

Standard

cubic-bezier(0.4, 0, 0.2, 1)

A curva padrão de tudo. Hover de cards, transições de cor, expansão de linhas. Rápida de sair, suave de chegar.

Enter (Decelerate)

cubic-bezier(0, 0, 0.2, 1)

Elementos que entram na tela — animações de scroll, modais que abrem, conteúdo que aparece.

Exit (Accelerate)

cubic-bezier(0.4, 0, 1, 1)

Elementos que saem — notificações, tooltips, elementos que desaparecem. Rápido e limpo.

CSS — Referência rápida

Transições prontas para uso

Hover de cor (200ms)

Para qualquer mudança de cor — texto, fundo, borda.

transition: color 0.2s ease,
background 0.2s ease,
border-color 0.2s ease;

Linha CTA (350ms)

Para o padrão de linha que expande nos botões ghost.

transition: width 0.35s
cubic-bezier(0.4, 0, 0.2, 1);

Card borda (300ms)

Border e pseudo-elemento bottom no hover de cards.

transition: border-color 0.3s ease;

/* pseudo ::after */
transition: transform 0.35s
cubic-bezier(0.4, 0, 0.2, 1);

Imagem zoom (400ms)

Scale sutil em imagens de produto — nunca acima de 1.08.

transition: transform 0.4s
cubic-bezier(0.4, 0, 0.2, 1);

/* hover */
transform: scale(1.05);

Fade-up de entrada (500ms)

Para elementos que entram no viewport — scroll ou page load.

@keyframes fadeUp {
from { opacity: 0;
transform: translateY(16px); }
to { opacity: 1;
transform: translateY(0); }
}
animation: fadeUp 0.5s ease both;

Stagger delay

Para listas de elementos em sequência — intervalo de 80ms.

.item:nth-child(1) { delay: 0s }
.item:nth-child(2) { delay: 0.08s }
.item:nth-child(3) { delay: 0.16s }
.item:nth-child(4) { delay: 0.24s }
